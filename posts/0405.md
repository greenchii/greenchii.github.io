## docker

### what is docker

AI时代老艺术家还在坚持手打笔记...

要把 Docker 讲清楚，最直观的比喻就是它的名字——**“集装箱”**（Docker 原意是码头工人）。

在没有集装箱之前，码头搬运工要处理散装的钢琴、木桶、甚至活的牲畜。每个人搬运的方式都不一样，很容易在运输过程中损坏，或者因为“环境不适应”导致钢琴走调、牲畜生病。

### 1. Docker 到底解决了什么问题？

最经典的代码报错场景是：**“在我的电脑上运行得好好的，怎么到你那儿（或服务器上）就崩了？”**

这通常是因为：

* 你的电脑装了 Python 3.10，服务器只有 Python 3.7。

* 你安装了一个特殊的系统补丁，服务器没有。

* 路径配置不一样，导致程序找不到文件。

**Docker 的出现，就是为了把你的程序和它运行所需要的所有“家当”：代码、运行时环境、库文件、配置等，全部打包进一个标准的“集装箱”（镜像）里。**

这个箱子无论扔到 Windows、Linux 还是云端服务器上，只要对方安装了 Docker 引擎，开箱即用，运行结果一模一样。

***

### 2. Docker 的三个核心概念

你可以通过这个逻辑链条来理解：

1. **Dockerfile（配方）**：一张纸，上面写着：我要一个 Ubuntu 系统，安装 Python 3.9，再把我的 `app.py` 拷贝进去。

2. **Image（镜像/集装箱本身）**：根据配方“刻录”出来的静态文件。它是只读的，就像一个安装包。

3. **Container（容器/运行中的集装箱）**：镜像运行起来后的实体。一个镜像可以启动无数个容器，它们之间互相隔离，互不干扰。

***

### 3. Docker 的实际应用场景

#### ① 一键搭建开发环境

以前你想学习 MySQL 数据库，得去官网下载、配置环境变量、解决安装报错。

现在用 Docker：

Bash

```plain&#x20;text
docker run --name my-mysql -e MYSQL_ROOT_PASSWORD=password -d mysql
```

一行命令，几秒钟后一个干净的数据库就跑起来了。不想用了直接删掉容器，**你的电脑主机依然干干净净，没有任何残留。**

#### ② 彻底解决环境隔离（不“打架”）

如果你要在同一台服务器上运行两个项目，一个需要旧版的 Java 8，一个需要新版的 Java 17。

* **传统方式**：配置极其痛苦，环境变量容易搞混。

* **Docker 方式**：项目 A 跑在 Java 8 的容器里，项目 B 跑在 Java 17 的容器里。它们就像住在不同的房间，完全不知道对方的存在。

#### ③ 秒级部署与横向扩展

当你的网站流量突然暴增，需要增加 10 台服务器。

* **以前**：运维人员得手动去每台机器装环境，忙到半夜。

* **现在**：直接把那个“集装箱”镜像分发到 10 台机器上，瞬间启动 10 个一模一样的服务。

***

### 4. Docker 与 虚拟机 (VM) 的区别

这是最容易混淆的地方。

* **虚拟机（如 VMware）**：是在你的电脑里**又模拟了一套完整的硬件和操作系统**。它很重，启动慢（几分钟），占用空间大（几 GB 甚至几十 GB）。

* **Docker**：它不模拟系统，而是**直接共用你主机的内核**。它非常轻量，启动只要几秒钟，占用的资源极小。

### 总结

**Docker 的本质就是“打包环境”。**

它让开发者不再为环境配置头疼，让运维人员不再为部署发愁。它把复杂的软件环境变成了像手机 App 一样：下载、运行、删除，整个过程标准化且互不干扰。

## 1.两数之和

给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出 **和为目标值&#x20;***`target`*  的那 **两个** 整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案，并且你不能使用两次相同的元素。

你可以按任意顺序返回答案。

https://leetcode.cn/problems/two-sum/description/?envType=study-plan-v2\&envId=top-100-liked

```c++
class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        unordered_map<int, int> map; // 值 -> 下标
        for (int i = 0; i < nums.size(); i++) {
        //从左到右开始遍历数组
            int complement = target - nums[i];//计算能和当前数相加得到target的数
            if (map.count(complement)) {
            //判断该数之前是否出现过（出现过会被存入map中）
                return {map[complement], i};//如果出现过就直接返回
            }
            map[nums[i]] = i;//没出现过，把当前数加入哈希表中
        }
        return {};
    }
};
```

## 49.字母异位词分组

给你一个字符串数组，请你将 字母异位词 组合在一起。可以按任意顺序返回结果列表。

**示例 1:**

**输入:** strs = \["eat", "tea", "tan", "ate", "nat", "bat"]

**输出:&#x20;**\[\["bat"],\["nat","tan"],\["ate","eat","tea"]]

**解释：**

* 在 strs 中没有字符串可以通过重新排列来形成 `"bat"`。

* 字符串 `"nat"` 和 `"tan"` 是字母异位词，因为它们可以重新排列以形成彼此。

* 字符串 `"ate"` ，`"eat"` 和 `"tea"` 是字母异位词，因为它们可以重新排列以形成彼此。

**示例 2:**

**输入:** strs = \[""]

**输出:&#x20;**\[\[""]]

**示例 3:**

**输入:** strs = \["a"]

**输出:&#x20;**\[\["a"]]

**<span style="color: rgb(216,57,49); background-color: inherit">核心思想：字母异位词排序后一定相同</span>**

<span style="color: rgb(216,57,49); background-color: inherit">比如 </span>`"eat"`<span style="color: rgb(216,57,49); background-color: inherit">、</span>`"tea"`<span style="color: rgb(216,57,49); background-color: inherit">、</span>`"ate"`<span style="color: rgb(216,57,49); background-color: inherit"> 排序后都是 </span>`"aet"`<span style="color: rgb(216,57,49); background-color: inherit">，所以可以用</span>**<span style="color: rgb(216,57,49); background-color: inherit">排序后的字符串</span>**<span style="color: rgb(216,57,49); background-color: inherit">作为 key，把相同的词归到一组。</span>

```c++
class Solution {
public:
    vector<vector<string>> groupAnagrams(vector<string>& strs) {
        unordered_map<string,vector<string>> map;//注意哈希表value是vector<string>
        for(string &s:strs){//&表示直接引用原来的，如果没有&的话需要复制
            string key=s;//key用于存放排序后的单词
            sort(key.begin(),key.end());
            map[key].push_back(s);//value存放原单词
            //这样哈希表直接就是分好组的结果
        }
        
        vector<vector<string>> res;
        for(auto &pair:map){
            res.push_back(pair.second);
        }
        return res;
    }
};
```

**`auto`** — 让编译器自动推断类型，不用你手写。这里 `map` 的每个元素类型是 `pair<string, vector<string>>`，写起来很长，用 `auto` 省事。

**`pair`** — 哈希表里每个元素都是一个键值对，长这样：

```plain&#x20;text
pair.first  = key，比如 "aet"
pair.second = value，比如 ["eat","tea","ate"]
```

**`pair.second`** — 就是取出 value，也就是同一组的字符串列表。

## 128.最长连续序列

给定一个未排序的整数数组 `nums` ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 `O(n)`*&#x20;*&#x7684;算法解决此问题。

**示例 1：**

**输入：**&#x6E;ums = \[100,4,200,1,3,2]
**输出：**&#x34;
**解释：**&#x6700;长数字连续序列是 \[1, 2, 3, 4]。它的长度为 4。

**示例 2：**

**输入：**&#x6E;ums = \[0,3,7,2,5,8,4,6,0,1]
**输出：**&#x39;

**示例 3：**

**输入：**&#x6E;ums = \[1,0,1,2]
**输出：**&#x33;

<span style="color: rgb(216,57,49); background-color: inherit">排序是 O(n </span> <span style="color: rgb(216,57,49); background-color: inherit">log</span> <span style="color: rgb(216,57,49); background-color: inherit"> n)，题目要求 O(n)，所以要换思路。</span>

**<span style="color: rgb(216,57,49); background-color: inherit">核心思想：用 </span>`unordered_set`**

<span style="color: rgb(216,57,49); background-color: inherit">对于每个数 </span>`n`<span style="color: rgb(216,57,49); background-color: inherit">，只有当 </span>`n-1`<span style="color: rgb(216,57,49); background-color: inherit"> </span>**<span style="color: rgb(216,57,49); background-color: inherit">不在</span>**<span style="color: rgb(216,57,49); background-color: inherit">集合里时，它才是某段连续序列的起点。从起点开始往后数，就能得到这段序列的长度。</span>

<span style="color: rgb(216,57,49); background-color: inherit">这样每个数只被访问常数次，整体 O(n)。</span>

```c++
class Solution {
public:
    int longestConsecutive(vector<int>& nums) {
        if(nums.empty())return 0;
        unordered_set<int> s(nums.begin(),nums.end());//去重，无序，O（1）
        int res=0;

        for(auto n:s){
            if(s.count(n-1))continue;//n-1存在，说明不是起点，跳过

            int cur=1;
            while(s.count(n+cur))cur++;
            res=max(res,cur);
        }
        return res;
    }
};
```

**unordered\_set\<int> s(nums.begin(), nums.end());&#x20;**

**等价于手动这样写：**

```c++
unordered_set<int> s;
for (int n : nums) {
    s.insert(n);  // 自动去重，插入重复值会被忽略
}
```

## Agent Skills

**Agent Skills** 是指 AI 智能体（Agent）能够调用的**能力模块**，让 Agent 不只是"聊天"，而是能真正**执行任务**。

***

**为什么需要 Skills？**

纯语言模型只能输入文字、输出文字，但现实任务需要：

* 搜索网页

* 执行代码

* 读写文件

* 调用 API

Skills 就是把这些能力封装成 Agent 可以调用的工具。

***

***

```plain&#x20;text
用户输入任务
```
